## Branches

Branches permitem desenvolver features de forma **isolada** sem afetar o código principal.

---

### Fluxo recomendado

```bash
# Criar e trocar para nova branch
git checkout -b feat/nova-funcionalidade

# Trabalhar, commitar...
git add .
git commit -m "Adiciona nova funcionalidade"

# Enviar branch para o remoto
git push -u origin feat/nova-funcionalidade

# Após o merge, limpar
git checkout main
git pull
git branch -d feat/nova-funcionalidade
```

---

### Branches protegidas

`main` — código em produção, **nunca** commitar direto
